## Task 1 — Fix `applyAICommands` bug in dashboard

**File:** `src/app/(routes)/dashboard/page.tsx`

**Problem:** The function wraps `setCapabilities` inside `setNodeStyles`, then returns `prevCaps` instead of `result.capabilities`. Structural changes from AI never reach the canvas.

**Fix:** Restructure so `setCapabilities` returns `result.capabilities` and `setNodeStyles` is called with `result.nodePatches` in the same pass.

```ts
// Current (broken)
const applyAICommands = useCallback((commands: DiagramCommand[]) => {
  setCapabilities((prevCaps) => {
    setNodeStyles((prevStyles) => {
      const result = executeCommands(commands, prevCaps, prevStyles);
      useCatalogStore.setState({ capabilities: result.capabilities, isDirty: true });
      setTimeout(() => setNodeStyles(result.nodePatches), 0);
      return prevStyles; // ← never updates nodeStyles
    });
    return prevCaps; // ← never updates capabilities
  });
}, []);

// Fixed
const applyAICommands = useCallback((commands: DiagramCommand[]) => {
  setCapabilities((prevCaps) => {
    const result = executeCommands(commands, prevCaps, nodeStyles);
    setNodeStyles(result.nodePatches);
    useCatalogStore.setState({ capabilities: result.capabilities, isDirty: true });
    return result.capabilities;
  });
}, [nodeStyles]);
```

## Task 4 — Implement `ADD_NODE` in executor

**File:** `src/lib/commands/executor.ts`

Add a new `case "ADD_NODE":` block. Logic:

1. If `parentId` is non-null, verify a capability with that ID exists — push to `errors` and break if not found.
2. Validate level: if `parentId` is non-null, the parent's `level` must equal `cmd.level - 1`. Push error and break on mismatch.
3. Compute `sort_order`:
   - If `insertAfterId` is provided, find that sibling's `sort_order` and use `sort_order + 1`.
   - Otherwise, use `max(siblings.sort_order) + 1`.
   - Shift sort_orders of all subsequent siblings up by 1 to avoid collisions.
4. Derive `catalog_id` from the parent (or any existing capability if `parentId` is null).
5. Push a new `Capability` object:

```ts
{
  id: cmd.tempId,
  catalog_id: /* from parent or sibling */,
  parent_id: cmd.parentId ?? null,
  level: cmd.level,
  name: cmd.name,
  description: cmd.description ?? null,
  note: null,
  sort_order: /* computed */,
  source: "ai_generated",
  is_deleted: false,
  created_at: new Date().toISOString(),
  updated_at: new Date().toISOString(),
}
```

6. Push `messages.push(\`Added "${cmd.name}"\`)`.

## Task 7 — Create `AIPanel` component

**File to create:** `src/components/canvas/AIPanel.tsx`

A collapsible strip docked to the bottom of the canvas area.

**States (internal to component):**
- `aiPrompt: string`
- `aiLoading: boolean`
- `aiSummary: string | null`
- `aiError: string | null`
- `isOpen: boolean` (collapsed by default)

**Props:**
```ts
interface AIPanelProps {
  capabilities: Capability[];
  nodeStyles: Record<string, NodeStylePatch>;
  onCommandsApplied: (commands: DiagramCommand[], summary: string) => void;
}
```

**Submit handler** calls `POST /api/transform` with `{ prompt, capabilities, nodeStyles }` and fires `onCommandsApplied(data.commands, data.summary)`.

**UI layout:**

```
┌──────────────────────────────────────────────────┐
│ ✨ AI Edit  [collapsed strip — click to expand]  │
├──────────────────────────────────────────────────┤  ← when open
│ [textarea: "Describe what to change..."]         │
│ [Submit button]  [spinner when loading]          │
│ Summary: "Renamed Finance to Financial Services" │  ← after success
│ Error: "..." (red)                               │  ← on failure
└──────────────────────────────────────────────────┘
```

**Prompt examples** shown as clickable chips when textarea is empty:
- "Make all L0 nodes dark blue"
- "Rename 'Finance' to 'Financial Services'"
- "Add L1 capability 'AI & Automation' under 'Technology'"
- "Delete the 'Legacy Systems' node and lift its children"
- "Add a note to 'Customer Experience' saying needs review"
