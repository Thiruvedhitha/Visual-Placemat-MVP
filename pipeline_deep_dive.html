Step 1 — Excel Upload & Parsing

STEP 1
Parse Excel → Flat Rows
src/lib/parser/excelParser.ts

SheetJS reads the .xlsx buffer. It finds the sheet named "Capability Catalog" (or uses the first sheet), reads the header row to locate L0–L3 columns by regex, then iterates over data rows.

INPUT — Excel file (3 rows shown)
L0 Capability NameL1 Capability NameL2 Capability NameL3 Capability NameDescription
Strategy ManagementPortfolio ManagementPortfolio Budget MgmtBudget DefinitionDefine budget...
Budget ForecastingForecast spend...
Portfolio PrioritizationPrioritize items...
→
OUTPUT — ParsedCapabilityRow[]
[ { l0: "Strategy Management", l1: "Portfolio Management", l2: "Portfolio Budget Mgmt", l3: "Budget Definition", description: "Define budget..." }, { l0: null, l1: null, l2: null, l3: "Budget Forecasting", description: "Forecast spend..." }, { l0: null, l1: null, l2: "Portfolio Prioritization", l3: null, description: "Prioritize items..." } ]
Key logic in the parser
  • Header row scanned with regex: /l0/i, /l1/i, /l2/i, /l3/i, /desc/i
  • Empty cells become null — the Excel has merged cells, so only the first row of a group has the L0/L1 value
  • Rows where ALL levels are null are skipped entirely
  • Output is a flat array — no tree structure yet, just raw rows

How parent_id is resolved — the key algorithm
// Phase 1: Walk rows, track current L0/L1/L2 context // Build records with full-path selfKey for uniqueness Row 1 → L0 appears → currentL0 = "Strategy Mgmt" L1 appears → currentL1 = "Portfolio Mgmt" L2 appears → currentL2 = "Budget Mgmt" L3 appears → record: selfKey = "3:Strategy Mgmt/Portfolio Mgmt/Budget Mgmt/Budget Definition" parentKey= "2:Strategy Mgmt/Portfolio Mgmt/Budget Mgmt" Row 2 → Only L3 → currentL0/L1/L2 unchanged (inherited from Row 1) L3 record: selfKey = "3:Strategy Mgmt/Portfolio Mgmt/Budget Mgmt/Budget Forecasting" parentKey= "2:Strategy Mgmt/Portfolio Mgmt/Budget Mgmt" ← same parent! Row 3 → Only L2 → currentL2 = "Prioritization" L2 record: selfKey = "2:Strategy Mgmt/Portfolio Mgmt/Prioritization" parentKey= "1:Strategy Mgmt/Portfolio Mgmt" // Phase 2: Insert level-by-level (L0 → L1 → L2 → L3) // After inserting L0, we have: pathToId["0:Strategy Mgmt"] = uuid-A // When inserting L1, look up parentKey "0:Strategy Mgmt" → uuid-A // When inserting L2, look up parentKey "1:Strategy Mgmt/Portfolio Mgmt" → uuid-B // etc.

Why full-path keys? If two L2s have the same name under different L1 parents (e.g., "Portfolio Budget Mgmt" under both "Portfolio Mgmt" and "Financial Mgmt"), simple "level:name" keys would collide. Full ancestor paths guarantee uniqueness.

Step 5 — Layout Engine (Flat Rows → X,Y Positions)

STEP 5
buildCanvasNodes() — Tree Build → Number → Position
src/lib/canvas/layoutEngine.ts

This is the core algorithm. It takes flat DB rows and produces positioned React Flow nodes. Three sub-steps:

5a. Build Parent-Child Tree

INPUT — Flat capability rows
uuid-A (L0) parent=null uuid-B (L1) parent=uuid-A uuid-C (L2) parent=uuid-B uuid-D (L3) parent=uuid-C uuid-E (L3) parent=uuid-C uuid-F (L2) parent=uuid-B
→
OUTPUT — TreeNode hierarchy
Strategy Mgmt (L0) └─ Portfolio Mgmt (L1) ├─ Budget Mgmt (L2) │ ├─ Budget Definition (L3) │ └─ Budget Forecasting (L3) └─ Prioritization (L2)

5b. Assign Hierarchical Numbers

INPUT — Tree (unnumbered)
Strategy Mgmt └─ Portfolio Mgmt ├─ Budget Mgmt │ ├─ Budget Definition │ └─ Budget Forecasting └─ Prioritization
→
OUTPUT — Numbered tree
1 Strategy Mgmt └─ 1.1 Portfolio Mgmt ├─ 1.1.1 Budget Mgmt │ ├─ 1.1.1.1 Budget Definition │ └─ 1.1.1.2 Budget Forecasting └─ 1.1.2 Prioritization

5c. Calculate X,Y Positions

Y-position calculation trace
// L0 at y=0 y = 0 ← L0 header y = 0 + 48(L0_H) + 3(V_GAP) = 51 ← L1 node y = 51 + 48(L1_H) + 3(V_GAP) = 102 ← L2: Budget Mgmt y = 102 + 48(L2_H) + 3(V_GAP) = 153 ← L3: Budget Definition y = 153 + 42(L3_H) + 3(V_GAP) = 198 ← L3: Budget Forecasting y = 198 + 42(L3_H) + 3(V_GAP) + 6(L2_GAP) = 249 ← L2: Prioritization (new L2 block)
